# DARFUS Jewellery ERP

واجهة ERP حديثة ومتجاوبة لإدارة الذهب والمجوهرات، مبنية باستخدام **Next.js 16 + TypeScript + Tailwind CSS + next-intl**.

## التشغيل

> يتطلب Node.js 20.9 أو أحدث.

```bash
npm install
npm run dev
```

ثم افتح:

- العربية: `http://localhost:3000/ar/login`
- English: `http://localhost:3000/en/login`

## بيانات الدخول التجريبية

```text
Email: admin@admin.com
Password: 123456
```

## إنشاء حساب جديد

من صفحة تسجيل الدخول اختر **إنشاء حساب جديد**، ثم أكمل المراحل التالية:

1. بيانات الحساب ومساحة العمل.
2. بيانات المستخدم والوظيفة وحجم الشركة.
3. عنوان الشركة والبيانات التجارية ورفع الشعار.
4. مراجعة البيانات وإنشاء الحساب.

الحساب، بيانات الشركة، الشعار، الجلسة والبيانات التجريبية تُحفظ محليًا في `localStorage` لتجربة النظام بدون Backend. الشعار يجب أن يكون صورة وألا يزيد حجمه على 1MB.

## أهم الخصائص

- حماية جميع صفحات لوحة التحكم وتحويل الزائر إلى تسجيل الدخول.
- تسجيل دخول تجريبي وإنشاء حساب متعدد المراحل يعملان فعليًا.
- العربية RTL والإنجليزية LTR من خلال `messages/ar.json` و`messages/en.json`.
- ظهور اسم الشركة وشعارها في لوحة التحكم والهيدر والقائمة الجانبية.
- قائمة جانبية قابلة للتصغير والتكبير مع حفظ حالتها.
- بحث عام في الهيدر داخل الأصول والعملاء والفواتير.
- مكوّن بحث وفلترة موحّد وقابل لإعادة الاستخدام في صفحات القوائم.
- صفحات المبيعات، نقطة البيع، المخزون، العملاء، الموردين، الموظفين، الحسابات، التقارير، التدقيق والإعدادات.
- إضافة بيانات وتحديثها محليًا، إنشاء فاتورة POS، رفع الشعار، تصدير CSV وإعادة ضبط بيانات العرض.
- Light/Dark mode مع تباين ألوان واضح.
- تصميم Responsive للكمبيوتر والتابلت والموبايل.

## الملفات المهمة

```text
app/[locale]/login/page.tsx       صفحة تسجيل الدخول
app/[locale]/signup/page.tsx      إنشاء الحساب متعدد المراحل
messages/ar.json                  الترجمة العربية
messages/en.json                  الترجمة الإنجليزية
contexts/auth-context.tsx         الحسابات والجلسة وبيانات الشركة
contexts/erp-context.tsx          بيانات النظام التجريبية
hooks/use-data-filters.ts         دوال البحث والفلترة القابلة لإعادة الاستخدام
components/ui/data-toolbar.tsx    واجهة البحث والفلاتر
components/layout/sidebar.tsx     القائمة الجانبية القابلة للتصغير
```

## أوامر الفحص

```bash
npm run typecheck
npm run build
npm start
```

## تنظيف البيانات المحلية

يمكن إعادة بيانات الـERP من صفحة الإعدادات. لحذف الحسابات والجلسة بالكامل، افتح Developer Tools ثم امسح Local Storage الخاص بالموقع.

## ملاحظة إنتاجية

هذه النسخة تنفذ التدفق المطلوب باستخدام التخزين المحلي. عند ربط Backend حقيقي، استبدل عمليات `localStorage` داخل `contexts` بخدمات API مع إبقاء الواجهات والمكونات كما هي.
